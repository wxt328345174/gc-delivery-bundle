# 安装说明

## 一、交付内容

交付包默认包含以下内容：

- `install.sh`：客户唯一需要执行的安装入口
- `customer.conf`：安装清单和少量参数配置
- `packages/`：离线安装包目录
- `.internal/`：内部脚本目录，客户通常不需要进入

## 二、安装前准备

请在目标设备上确认以下条件：

- 操作系统为 `openEuler aarch64`
- 使用 `root` 用户执行安装
- 系统具备 `dnf`、`rpm`、`systemctl`
- 交付包已经完整复制到目标设备本地目录

## 三、如何修改配置

编辑同目录下的 `customer.conf`。

常见示例：

只做基础环境准备，不安装软件：

```bash
SOFTWARES=()
```

只安装 runtime：

```bash
SOFTWARES=("runtime")
```

安装 chromium 和向日葵：

```bash
SOFTWARES=("chromium" "sunlogin")
```

安装 Qt Creator：

```bash
SOFTWARES=("qt")
```

说明：

- `REBOOT_POLICY="prompt"`：安装结束后提示是否重启
- `REBOOT_POLICY="always"`：安装结束后自动重启
- `REBOOT_POLICY="never"`：安装结束后不自动重启

如果需要安装 Qt Creator，请先确认 `packages/qt-creator-4.15.2-openeuler-aarch64.tar.gz` 已放入 `packages/` 目录。该方式为预编译安装，客户侧不需要编译。

## 四、如何执行

进入交付目录后执行：

```bash
bash install.sh
```

如果只想先检查流程和安装包，不真正安装：

```bash
bash install.sh --dry-run
```

如果配置文件不在默认位置，可以指定：

```bash
bash install.sh --config ./customer.conf
```

## 五、安装后检查

### 1. runtime

安装完成并重启后，可以检查：

```bash
ps aux | grep runtime
systemctl status ethercat
```

### 2. chromium

重启或重新登录图形界面后：

- 使用配置中的普通用户登录
- 从桌面菜单打开 Chromium

### 3. 向日葵

安装完成后，可检查：

```bash
rpm -qa | grep -i sunlogin
```

### 4. Qt Creator

安装完成后，可检查：

```bash
qtcreator --help
```

如果设备带图形桌面，也可以从应用菜单打开 Qt Creator。

## 六、常见问题

### 1. 提示必须使用 root

请切换到 `root` 后重新执行：

```bash
sudo -i
bash install.sh
```

### 2. 提示缺少安装包

请确认 `packages/` 目录中的文件没有被删除或改名。

### 3. chromium 未设置密码

如果 `customer.conf` 中 `CHROMIUM_PASSWORD=""`，脚本会在执行时要求输入密码；如果跳过了输入，后续可以手动执行：

```bash
passwd gcuser
```

### 4. 安装完成后未重启

某些软件需要重启后才能完全生效，特别是 `runtime`、图形自动登录和 Qt 桌面菜单相关配置。
