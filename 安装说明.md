# 安装说明

## 一、交付内容

交付包默认包含以下内容：

- `install.sh`：客户唯一需要执行的安装入口
- `customer.conf`：客户安装清单和少量参数配置
- `packages/`：离线安装包目录
- `.internal/`：内部脚本目录，客户通常不需要进入

## 二、安装前准备

请在目标设备上确认以下条件：

- 操作系统为 `openEuler 22.04 aarch64`
- 使用 `root` 用户执行安装
- 系统具备 `dnf`、`rpm`、`systemctl`
- 交付包已经完整复制到目标设备本地目录

## 三、如何修改配置

编辑同目录下的 `customer.conf`：

```bash
SOFTWARES=("runtime" "chromium" "sunlogin")
REBOOT_POLICY="prompt"
```

常见配置说明：

- 只安装基础环境，不安装软件：
```bash
SOFTWARES=()
```

- 只安装 runtime：
```bash
SOFTWARES=("runtime")
```

- 只安装 chromium 和向日葵：
```bash
SOFTWARES=("chromium" "sunlogin")
```

- `REBOOT_POLICY="prompt"`：执行结束后提示是否重启
- `REBOOT_POLICY="always"`：执行结束后自动重启
- `REBOOT_POLICY="never"`：执行结束后不重启，由人工处理

## 四、如何执行

进入交付目录后执行：

```bash
bash install.sh
```

如果只想先检查流程和安装包，不真正安装：

```bash
bash install.sh --dry-run
```

如果配置文件不在默认位置，可指定：

```bash
bash install.sh --config ./customer.conf
```

## 五、安装后检查

### 1. runtime

安装完成并重启后，可检查：

```bash
ps aux | grep runtime
systemctl status ethercat
```

如果能看到 `plc-runtime` 相关进程，说明 runtime 基本正常。

### 2. chromium

重启或重新登录图形界面后：

- 使用配置中的普通用户登录
- 从桌面菜单中打开 Chromium

### 3. 向日葵

安装完成后，可在系统菜单中查看向日葵是否已出现，或使用以下命令检查：

```bash
rpm -qa | grep -i sunlogin
```

## 六、常见问题

### 1. 提示必须使用 root

请切换到 `root` 用户后再执行：

```bash
sudo -i
bash install.sh
```

### 2. 提示缺少安装包

请确认 `packages/` 目录中的文件没有被删除或改名。

### 3. chromium 未设置密码

如果 `customer.conf` 中 `CHROMIUM_PASSWORD=""`，脚本会在执行时要求输入密码；如果未输入，则需要后续手动执行：

```bash
passwd gcuser
```

### 4. 安装完成后未重启

某些软件需要重启后才能完全生效，特别是 `runtime` 和图形登录相关配置。
